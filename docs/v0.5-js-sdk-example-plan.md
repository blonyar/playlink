# Playlink v0.5 JS SDK & Example Plan

## 1. Stage Goal

v0.5 turns the stable room server into something easier for game prototypes to consume.

The smallest useful loop is:

```text
Start server -> import JS helper -> connect two players -> create/join room -> exchange game-shaped messages
```

This stage should prove that the existing WebSocket JSON protocol, room lifecycle, Web Console, and LAN discovery metadata are enough to build a small client-facing workflow.

## 2. Scope

### In Scope

- A tiny JavaScript client helper in `examples/js-client/playlink-client.js`
- A runnable SDK-style demo script in `examples/js-client/sdk-demo.js`
- A tiny browser mini-game example in `examples/js-client/mini-game.html`
- Documentation for using the helper and running the demos
- Reuse of the existing v0.3 protocol and v0.4 server metadata/discovery flow
- Lightweight examples of room messages that look like game events

### Out of Scope

Do not implement these in v0.5:

- relay mode
- P2P/NAT traversal
- accounts or authentication
- global matchmaking
- production admin permissions
- package publishing
- a full engine SDK
- game-specific server simulation

## 3. Current Baseline

Already available before v0.5:

- Rust Axum server
- WebSocket endpoint `/ws`
- HTTP endpoints `/health`, `/api/rooms`, `/api/rooms/:room_id`, and `/api/server`
- structured JSON protocol
- create, join, leave, list, inspect rooms
- room broadcasts
- structured error codes
- ping/pong and idle timeout
- Web Debug Console
- LAN discovery prototype
- JavaScript smoke, negative, idle-timeout, and discovery scripts

## 4. JS Helper Shape

`examples/js-client/playlink-client.js` exports `PlaylinkClient`.

The helper intentionally stays small and example-oriented. It is not a published package yet.

Supported operations:

```js
import { PlaylinkClient } from './playlink-client.js';

const alice = new PlaylinkClient({ name: 'alice' });
await alice.connect();

const roomId = await alice.createRoom({ roomName: 'demo', maxPlayers: 4 });
await alice.joinRoom(roomId, 'alice');
alice.sendRoomMessage({ kind: 'move', x: 3, y: 7 });
```

Important behavior:

- request/response messages use the protocol `id` field
- server errors reject the matching request with `error.code`
- room events are available through `waitFor(type, predicate)` or `on(type, handler)`
- HTTP helpers call `/api/rooms` and `/api/server`

## 5. Demo Script

`examples/js-client/sdk-demo.js` demonstrates:

1. Connect Alice and Bob to `/ws`.
2. Read `/api/server` metadata.
3. Alice creates a room.
4. Alice and Bob join the room.
5. Alice sends a chat-style room event.
6. Bob sends a movement-style room event.
7. The script verifies both broadcasts.
8. Bob leaves and Alice receives `player_left`.

Run it with:

```bash
npm --prefix examples/js-client run sdk-demo
```

## 6. Mini Game Browser Example

`examples/js-client/mini-game.html` is a tiny browser example for game-prototype usage.

It demonstrates:

1. Connecting to the Playlink WebSocket endpoint.
2. Creating or joining a room.
3. Adding an optional local bot so one browser tab can show multiplayer behavior.
4. Sending `move` events as room messages.
5. Rendering local and remote player positions.
6. Smoothing local input with `requestAnimationFrame`.
7. Broadcasting position at about 20Hz instead of on every key event.
8. Interpolating remote player positions.
9. Leaving the room and reacting to `player_left`.

Run it with:

```bash
npm --prefix examples/js-client run mini-game
```

Then open:

```text
http://127.0.0.1:7780/
```

Open one browser tab, connect, create and join a room, then click `Add Bot` to spawn a second local client. You can also open two browser tabs, connect both, create a room in one tab, copy the room ID into the other tab, and move with arrow keys or WASD.

## 7. LAN Discovery Relationship

The SDK demo uses `PLAYLINK_WS_URL` and `PLAYLINK_HTTP_URL` environment variables, so it can connect to a server found manually or through the v0.4 discovery script.

Suggested LAN flow:

```bash
PLAYLINK_LAN_DISCOVERY=1 rustup run stable cargo run
npm --prefix examples/js-client run discover-lan
PLAYLINK_WS_URL=ws://192.168.1.20:7777/ws PLAYLINK_HTTP_URL=http://192.168.1.20:7777 npm --prefix examples/js-client run sdk-demo
```

The discovery script remains separate from the SDK helper for now. v0.5 should not hide network/firewall behavior behind magic connection logic.
